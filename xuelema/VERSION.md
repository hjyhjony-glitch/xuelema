# 学了吗APP 版本管理规范

## 📋 版本号规则

**格式**: `主版本.次版本.修订版本` (Semantic Versioning)

```
版本号示例: 1.0.0

- 主版本 (1): 重大功能变更或架构调整
- 次版本 (0): 新功能添加
- 修订版本 (0): Bug修复或小改进
```

**版本命名**:
- `1.0.0` - 首个正式版本
- `1.1.0` - 添加新功能
- `1.1.1` - Bug修复
- `2.0.0` - 重大重构

## 🔄 版本更新流程

### 步骤1: 更新版本号
编辑 `pubspec.yaml`:
```yaml
version: 1.1.0  # 更新版本号
```

### 步骤2: 记录变更
更新 `CHANGELOG.md`:
```markdown
## [1.1.0] - 2026-02-18

### 新增
- xxx功能

### 修复
- xxx问题

### 优化
- xxx改进
```

### 步骤3: 提交代码
```bash
git add -A
git commit -m "feat: version 1.1.0 - 新增功能描述"
git push
```

### 步骤4: 打标签
```bash
git tag v1.1.0
git push origin v1.1.0
```

## ✅ 版本发布检查清单

发布新版本前需确认：

- [ ] 更新 `pubspec.yaml` 版本号
- [ ] 更新 `CHANGELOG.md`
- [ ] 运行所有测试 (`flutter test`)
- [ ] 构建验证 (`flutter build apk --debug`)
- [ ] 代码审核通过 (ARC)
- [ ] 更新文档
- [ ] 创建 Git Tag
